# Portfo: Сайт как продукт

Сайт портфолио в 2026 году не должен быть статичным резюме. Это инструмент, который доказывает твою экспертизу в реальном времени.

## 🔴 Проблема
Большинство портфолио быстро устаревают:
1. **Статика**: Данные об активности (GitHub) приходится обновлять вручную.
2. **Плохое SEO**: Одностраничные сайты часто не индексируются по ключевым навыкам.
3. **Отсутствие глубины**: Список названий проектов без разбора проблем и решений не дает рекрутеру контекста.

## ⚡️ Решение: Максимальная автоматизация
Я собрал этот сайт за 3 дня, внедрив в него инженерные практики, которые обычно используются в больших продуктах.

### 1. Нативный i18n (Astro 5)
Вместо JS-подмены контента, сайт использует физические роуты `/ru/` и `/en/`. Это гарантирует 100% индексацию всех версий сайта поисковиками.

### 2. Динамическая статистика
Интеграция с GitHub API через систему кеширования. График активности и данные профиля обновляются автоматически при каждой сборке сайта.

### 3. Структурированные кейсы (MDX)
Каждый проект - это глубокий разбор с использованием MDX, что позволяет гибко управлять версткой, вставлять интерактивные компоненты и диаграммы.

## 🛠 Технологический стек
- **Astro 5**: Для генерации максимально легкого и быстрого статического HTML.
- **TypeScript**: Для типизации контента и данных проектов.
- **GitHub Actions**: Автоматический деплой и пре-варминг кеша данных.
- **Linear**: Для управления бэклогом задач по развитию сайта.

---

> **Insights:** Хорошее портфолио должно говорить о тебе даже тогда, когда ты спишь. Автоматизация данных и качественный контент превращают сайт в твоего лучшего "менеджера по продажам".
